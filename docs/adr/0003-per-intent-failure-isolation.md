# Per-intent failure isolation on tool failure

`lookup_booking` is shared across booking-change, upsell, and some property intents, and is deliberately flaky. The naive approach — escalate the whole turn if any tool call fails — would mean a guest loses their wifi-password answer just because an unrelated booking lookup happened to fail. We decided failure isolation happens per intent: only intents that actually depend on the failed tool's result escalate; independent intents (e.g. FAQ answered via `answer_from_notes`) still resolve and appear in the same consolidated reply. This requires "consolidate" to track, per intent, which tool results it actually depends on — a small bookkeeping cost in exchange for not defeating the automation's purpose (easy questions answered immediately, only genuinely hard/failed ones escalate).
