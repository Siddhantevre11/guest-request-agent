# Fact rendering via deterministic template slots, not LLM free-text

The tools already guarantee facts come from real data, so "re-check facts against DB" isn't protecting against invented data — it's protecting against the *drafting* LLM garbling good data in prose (e.g. tool returns 11am checkout, draft says "stay until 1pm"). Rather than parse factual claims back out of free text (itself a fuzzy problem), we keep high-consequence facts deterministic: the reply template has explicit slots for checkout time, price, dates, etc., filled directly from tool-returned values. The LLM writes the natural connective language around the slots but never hand-types a consequential number. On this build, "verify" therefore means the facts are rendered deterministically by construction, not extracted-and-compared after the fact. Free-prose fact extraction/comparison is deferred (production hardening).
