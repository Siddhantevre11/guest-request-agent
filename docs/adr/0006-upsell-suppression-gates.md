# Upsell suppression gates

An upsell is agent-initiated (the guest didn't ask), so it needs guardrails beyond "a gap night exists." We decided deterministic code gates the upsell on three conditions, all of which must hold before an offer is made: (1) `find_upsell` returns a real gap from actual calendar data — never invented; (2) memory shows the upsell was not already offered/declined for this conversation — prevents nagging repeat offers; (3) the current message carries no complaint/negative intent — never upsell someone who just reported a problem (e.g. broken AC). Conditions (2) and (3) reuse machinery already in hand (the structured memory summary and the intent classification with sentiment), so they're suppression rules, not new subsystems. The LLM may surface that a gap exists; deterministic code decides whether it's actually offered.
